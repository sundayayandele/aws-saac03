🔐 1. Design Secure Architectures (30% of exam)

Security is now the heaviest-weighted domain. Every architecture must begin with least privilege and private connectivity.

🔑 IAM – Identity & Access Management

  • IAM Roles over long‑term access keys. Use roles for EC2, Lambda, and cross‑account access.
  • IAM Policies: JSON structure with Effect, Action, Resource. Prefer managed policies where possible.
  • IAM Identity Center (successor to AWS SSO) for workforce identity federation.
  • SCPs (Service Control Policies) for AWS Organizations – guardrails across accounts.

🔒 Encryption & Key Management

  • KMS (AWS Key Management Service): Symmetric vs asymmetric keys, automatic key rotation.
  • Encryption at rest: S3 (SSE‑S3, SSE‑KMS, DSSE‑KMS), EBS, RDS, DynamoDB.
  • Encryption in transit: TLS everywhere; use AWS Certificate Manager (ACM) for free public certs.
  • CloudHSM for dedicated hardware security modules (compliance-heavy workloads).

🌐 Network Security – VPC

  • Security Groups (stateful) vs Network ACLs (stateless).
  • VPC Endpoints (Gateway for S3/DynamoDB, Interface for others) to keep traffic within AWS.
  • AWS Network Firewall & WAF for application‑layer protection.
  • AWS Shield (Standard automatic, Advanced for DDoS protection).
💡 Exam Tip: If a scenario allows private access (VPC Endpoint) over public internet, the private option is almost always correct.

⚡ 2. Design Resilient Architectures (26%)

🔄 High Availability & Fault Tolerance

  • Multi‑AZ deployments for RDS, Aurora, ElastiCache, and EC2 Auto Scaling across AZs.
  • Load Balancers: ALB (Layer7, path/host routing), NLB (Layer4, static IP), GWLB (appliance).
  • Route 53 routing policies: Simple, Weighted, Latency, Failover, Geolocation.

📦 Decoupling & Messaging

  • SQS (Standard vs FIFO) + SNS for pub/sub + EventBridge for event buses.
  • AWS Step Functions for serverless workflows with error handling.

📈 3. Design High-Performing Architectures (24%)

💻 Compute & Scaling

  • EC2 Auto Scaling with launch templates, lifecycle hooks, and scaling policies (target tracking, step, scheduled).
  • AWS Lambda concurrency, provisioned concurrency, and Lambda@Edge for global low latency.
  • Elastic Load Balancing: ALB supports gRPC, WebSockets; NLB preserves source IP.

🗂️ Database & Caching

  • RDS read replicas (up to 15 for Aurora), Multi‑AZ for HA.
  • ElastiCache (Redis vs Memcached) – Redis for persistence, replication, pub/sub.
  • DynamoDB – on‑demand vs provisioned, DAX for microsecond reads.

💰 4. Design Cost‑Optimized Architectures (20%)

  • EC2 pricing: On‑Demand, Reserved (Standard/Convertible), Savings Plans, Spot Instances.
  • S3 Storage Classes: S3 Standard → Intelligent‑Tiering → Infrequent Access → Glacier → Deep Archive.
  • AWS Budgets & Cost Explorer to track spend.
  • Compute Optimizer for right‑sizing recommendations.
  • Trusted Advisor cost optimization checks.

🧠 2026 Exam Mindset & Final Tips

  • ✅ Always choose private connectivity when possible.
  • ✅ Decouple applications with SQS, SNS, EventBridge.
  • ✅ Use managed services to reduce operational overhead (Aurora, DynamoDB, Lambda).
  • ✅ Read the question carefully – distractors are often technically correct but miss a key requirement.
