01

KMS — AWS Key Management Service

A fully managed service for creating, rotating, and controlling cryptographic keys. Integrates natively with most AWS services and provides a centralized audit trail via CloudTrail.

💡
Envelope Encryption

KMS never encrypts large datasets directly. It generates a Data Encryption Key (DEK), encrypts your data with the DEK locally, then KMS encrypts the DEK itself. The plaintext DEK is discarded — only the encrypted DEK is stored alongside the data.

KMS Key Types

TypeCreatorControllerRotationCost
AWS Managed Key AWS AWS Every 3 years (auto) Free
Customer Managed Key (CMK) Customer Customer Optional (annual / on-demand) $1/key/mo
AWS Owned Key AWS AWS (shared) AWS managed Free
02

Symmetric vs Asymmetric Keys

🔑 Symmetric Keys
  • Algorithm: AES-256-GCM
  • Single secret key for encrypt & decrypt
  • Default for AWS service integrations
  • Faster performance
  • Key never leaves KMS
  • APIs: Encrypt, Decrypt, GenerateDataKey
  • Used with: S3, EBS, RDS, DynamoDB, Secrets Manager
vs
🔐 Asymmetric Keys
  • Algorithms: RSA, ECC (ECDSA, ECDH)
  • Public/Private key pair
  • Encrypt with public → Decrypt with private (RSA)
  • Sign with private → Verify with public
  • Public key can be downloaded
  • Private key stays in KMS
  • Used for: cross-account, code signing, external clients
⚠️
Exam Trap

KMS symmetric keys are suitable when both encrypt and decrypt operations happen within AWS. Use asymmetric when external parties need to encrypt with a public key or verify signatures without AWS credentials.

03

Automatic Key Rotation

✅ Rotation Supported

  • Symmetric CMKs (annual or on-demand)
  • AWS managed keys (every 3 years)

❌ Rotation NOT Supported

  • Asymmetric keys
  • HMAC keys
  • Keys with imported key material
  • CloudHSM custom key store keys
💡
Critical Fact

Rotation changes only the backing key material. The Key ID and ARN remain the same — no application changes needed. Old key material is retained indefinitely so previously encrypted data can still be decrypted.

S3 Encryption at Rest

MethodKey ManagementKey ControlAudit TrailUse Case
SSE-S3 S3 manages automatically AWS No Simplest; no compliance req.
SSE-KMS KMS (CMK or AWS managed) Customer CloudTrail Audit + access control needed
DSSE-KMS Two KMS layers independently Customer CloudTrail Strongest S3 option; CNSSI 1253
SSE-C Customer provides key per request Customer No Full key custody (key NOT stored by AWS)

Enforcing Encryption

Add a bucket policy to deny any s3:PutObject request missing the encryption header:

{
  "Effect": "Deny",
  "Action": "s3:PutObject",
  "Condition": {
    "StringNotEquals": {
      "s3:x-amz-server-side-encryption": "aws:kms"
    }
  }
}

Or use S3 Default Encryption to automatically apply SSE-S3 or SSE-KMS to new objects without a bucket policy.

05

EBS & RDS Encryption

EBS

  • AES-256 via KMS
  • Encrypts: data at rest, data in transit to instance, snapshots
  • Enable per-volume or account-level default
  • Snapshots of encrypted volumes → encrypted
  • To encrypt existing: snapshot → copy with encrypt → new volume

RDS

  • Must enable at creation time
  • Cannot add encryption to existing DB
  • Encrypts: storage, backups, read replicas, snapshots
  • To encrypt existing: snapshot → copy with encrypt → restore
  • Read replicas inherit parent's encryption key
🚨
Exam Critical

Both EBS and RDS encryption cannot be enabled on existing resources. You must go through a snapshot > copy (with encryption enabled) > restore workflow.

DynamoDB Encryption

DynamoDB encrypts all tables at rest by default (since 2018). You cannot disable it. Three key options:

Customer Managed Key (CMK) Customer Full audit trail KMS costs apply
07

Encryption in Transit — TLS

All data moving between clients and AWS services, and between AWS services, must use TLS 1.2 minimum (TLS 1.3 preferred). AWS service endpoints are HTTPS by default.

ServiceHow to Enforce TLS
S3 Bucket policy: deny aws:SecureTransport: false
RDS / Aurora Parameter group: rds.force_ssl = 1
ALB HTTPS listener + security policy (e.g., ELBSecurityPolicy-TLS13-1-2)
API Gateway HTTPS only (no HTTP option for REST APIs)
ElastiCache Enable in-transit encryption at cluster creation
08

AWS Certificate Manager (ACM)

Public Certs (Free)

  • Free SSL/TLS certificates for AWS services
  • Auto-renews before expiry
  • Works with: ALB, NLB, CloudFront, API Gateway, Beanstalk
  • DNS or Email validation for domain ownership
  • Private key cannot be exported

ACM Private CA (Paid)

  • Issue private certs for internal services
  • ~$400/month per CA + per cert fees
  • Certificates CAN be exported (for use outside AWS)
  • Full PKI hierarchy management
  • Required for IoT, microservices mTLS
⚠️
CloudFront Region Requirement

ACM certificates are regional. CloudFront requires certs to be provisioned in us-east-1 (N. Virginia) regardless of where your origin is located.

09

CloudHSM

Dedicated, single-tenant Hardware Security Modules in the AWS cloud. You have exclusive access — AWS provides the hardware but has zero access to your keys.

FeatureAWS KMSCloudHSM
Hardware Shared (multi-tenant) Dedicated (single-tenant)
FIPS 140-2 Level Level 2 Level 3
AWS Access to Keys AWS manages keys Zero AWS access
Integration Native AWS (IAM policies) Manual (PKCS#11, JCE, CNG)
High Availability Built-in Must deploy 2+ HSMs in a cluster
Cost Per API call ~$1.45/hr per HSM

CloudHSM Use Cases

  • FIPS 140-2 Level 3 compliance requirements
  • Custom key management (AWS has no access)
  • Oracle TDE (Transparent Data Encryption)
  • SSL/TLS offloading on web servers
  • Digital signing with full key custody
  • Government / financial workloads
🎯
Exam Trigger Words for CloudHSM

"dedicated HSM" · "FIPS 140-2 Level 3" · "customer-managed keys with no AWS access" · "compliance requiring exclusive key control" · "Oracle TDE" → Answer is CloudHSM.

Decision Tree

Need to encrypt data in AWS?
│
├─ At rest in S3?
│   ├─ No key management needed ──────────────────── SSE-S3
│   ├─ Audit trail + access control ──────────────── SSE-KMS
│   └─ Maximum security / dual layer ─────────────── DSSE-KMS
│
├─ At rest in EBS / RDS / DynamoDB?
│   └─ Enable KMS encryption (CMK for audit + control)
│
├─ In transit?
│   ├─ Public-facing HTTPS ───────── ACM (free public certs)
│   └─ Enforce TLS ──────────────── Bucket policy / param group
│
└─ Key management?
    ├─ Standard workloads ──────────────────── AWS KMS (CMK)
    ├─ Fast, AWS-integrated, symmetric ──────── Symmetric CMK
    ├─ Cross-region signing / external ──────── Asymmetric CMK
    └─ FIPS L3 / dedicated HW / compliance ──── CloudHSM

Exam Quick Facts

01 KMS symmetric keys never leave KMS — envelope encryption is always used
02 Key rotation changes key material only — Key ID/ARN stays the same
03 Asymmetric: public key can be downloaded; private key stays in KMS

07 ACM public certs are free but cannot be exported

10 CloudHSM: AWS has zero access to your keys
