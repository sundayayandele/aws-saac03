01

Security Groups vs Network ACLs

🛡 Security Groups
  • Stateful — return traffic auto-allowed
  • Attached to EC2 instances / ENIs
  • Allow rules only — cannot explicitly deny
  • All rules evaluated simultaneously
  • Multiple SGs per instance allowed
  • Supports self-referencing (allow same-SG traffic)
  • Default: no inbound, all outbound allowed
🧱 Network ACLs
  • Stateless — must allow both directions explicitly
  • Applied at subnet level
  • Allow AND Deny rules supported
  • Rules processed in ascending number order
  • One NACL per subnet; one subnet per NACL
  • Must allow ephemeral ports 1024–65535 outbound
  • Default (new VPC): all traffic allowed

02

Stateful vs Stateless Explained

Stateful — Security Groups

Allow inbound port 443 → response traffic is automatically allowed outbound. No outbound rule needed for replies.

Inbound:  ALLOW TCP 443 0.0.0.0/0
Outbound: (no rule needed — auto-tracked)
Stateless — NACLs

Each packet evaluated independently. Must explicitly allow both the request AND the response ephemeral ports.

Inbound:  ALLOW TCP 443
Outbound: ALLOW TCP 1024-65535 ← required!
🚨
Exam Trap

Forgetting to add an outbound NACL rule for ephemeral ports (1024–65535) is the #1 NACL exam mistake. HTTP/S responses flow back on these dynamic ports — without an outbound allow rule, all responses are silently dropped.

03

NACL Rule Evaluation Order

Rules are evaluated in ascending numerical order. First match wins — processing stops immediately.

💡
Exam Tip

To block a specific IP, place a DENY rule with a lower number than the corresponding ALLOW rule. In the table above, rule 95 blocks an SSH attacker before rule 100+ can allow them. Security Groups cannot do this — only NACLs support explicit DENY.

04

VPC Endpoints

VPC Endpoints allow your resources to reach AWS services privately — no internet gateway, NAT device, VPN, or Direct Connect needed. Traffic stays on the AWS backbone, never touching the public internet.

🔒
Security Benefit

Endpoints eliminate the attack surface from NAT Gateways and internet-facing routes. Combined with endpoint policies, you can restrict which resources in your VPC can access which AWS service resources.

05

Gateway Endpoints

Gateway Endpoint
  • Services: S3 and DynamoDB only
  • Implemented via route table entry
  • Cost: Free
  • Region-scoped (not AZ-specific)
  • High availability built-in
  • Secured with Endpoint Policies
  • No DNS change required
  • ❌ Not accessible from on-premises
Example: S3 Gateway Endpoint Policy
{
  "Effect": "Allow",
  "Principal": "*",
  "Action": "s3:GetObject",
  "Resource": "arn:aws:s3:::bucket/*"
}
06

Interface Endpoints (PrivateLink)

Interface Endpoint
  • Services: All other AWS services (KMS, SSM, SNS, SQS, EC2 API…)
  • Creates an ENI with private IP in your subnet
  • Cost: Hourly + data processing
  • AZ-specific — deploy per-AZ for HA
  • Secured with Security Groups + Endpoint Policies
  • Private DNS enabled by default
  • ✅ Accessible from on-premises via VPN/DX

Exam Tip

If the scenario involves on-premises servers accessing an AWS service privately via Direct Connect or VPN → it must be an Interface Endpoint. Gateway Endpoints are not reachable outside the VPC.

07

AWS Network Firewall

A managed stateful/stateless network firewall for VPC-level protection. Inspects and filters traffic at Layers 3, 4, and some Layer 7 (domain filtering).

FeatureDetail
PlacementDedicated firewall subnet in each AZ
Rule typesStateless (fast), Stateful (connection-aware), Domain lists, Suricata IPS
ProtocolsAny TCP/UDP — not limited to HTTP
Multi-accountManaged centrally via AWS Firewall Manager
LoggingS3, CloudWatch Logs, Kinesis Firehose
Use casesOutbound domain filtering, C2 callback blocking, east-west IDS/IPS
08

AWS WAF

Layer 7 Web Application Firewall. Inspects HTTP/HTTPS traffic for application-layer attacks.

Supported Resources
  • Amazon CloudFront distributions
  • Application Load Balancers (ALB)
  • API Gateway (REST & HTTP APIs)
  • AWS AppSync (GraphQL)
  • Amazon Cognito user pools
Common WAF Rules
  • AWS Managed Rules (OWASP Top 10)
  • Rate-based — block IPs exceeding X req/5min
  • Geographic match — block by country
  • SQL injection & XSS detection
  • IP allow/block sets
  • Custom regex pattern matching
💡
Network Firewall vs WAF

WAF = Layer 7 HTTP/HTTPS web app protection (SQL injection, XSS, bot, rate limiting). Network Firewall = Layer 3/4 VPC traffic (any protocol, IDS/IPS, domain filtering, east-west). They complement each other — use both in a full defense-in-depth stack.

09

AWS Shield

Shield Standard
  • Free, automatic — no action required
  • Enabled for all AWS accounts
  • Protects against common L3/L4 DDoS: SYN floods, UDP reflection, DNS amplification
  • Integrated with CloudFront and Route 53
  • No visibility into attacks
  • No DRT access
Shield Advanced
  • $3,000/month per organization
  • Protects: EC2 EIPs, ELB, CloudFront, Global Accelerator, Route 53
  • DRT — 24/7 AWS DDoS Response Team access
  • Cost protection — AWS credits for scaling costs during attacks
  • Near real-time attack visibility + event summaries
  • Automatic WAF rule creation during L7 attacks
  • Health-based detection using Route 53
